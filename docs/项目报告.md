# 新加坡 HDB 组屋转售价格分析与预测系统 — 项目报告

---

## 一、项目背景

新加坡 Housing Development Board（HDB）组屋承载了超过 80% 新加坡常住人口的居住需求。组屋转售市场价格受房型、面积、剩余年限、地段、楼层等多重因素影响，对购房者而言，如何在海量交易数据中快速获取有效信息、合理评估房产价值、制定明智的购房决策，是一个具有实际意义的问题。

本项目基于 **data.gov.sg** 官方开放的 HDB 转售交易数据，选取 **Punggol、Sengkang、Hougang** 三个相邻的东北部镇区作为研究对象，构建了一个基于 Streamlit 的交互式分析与预测系统，帮助用户直观了解市场全貌、量化各因素影响、预测转售价格，并基于数据驱动的方法提出购房策略建议。

## 三、系统功能

系统共包含 **6 个功能页面**，通过 Streamlit 侧边栏导航：

### 1. 数据概览（01_overview.py）
- **多条件筛选**：镇区、年份、房型、面积、价格、剩余年限、楼层范围
- **统计看板**：交易套数、均价、中位数、总成交额、最高/最低价、最高/最低单价（8 指标卡片）
- **镇区分组统计**：各镇区套数、均价、中位数、单价、成交总额
- **趋势图表**：月度均价走势（折线图）、季度交易量（柱状图）
- **分布分析**：价格分布直方图、镇区×房型均价热力图
- **数据表格**：可配置列的交互式数据表，支持排序和 CSV 导出

### 2. 地图可视化（02_map.py）
- **镇区均价地图**：pydeck ScatterplotLayer 展示各镇区均价，圆圈大小 = 交易量，颜色深度 = 均价
- **热力分布地图**：基于 pydeck HeatmapLayer 的房价热力分布（含坐标抖动仿真）
- **时空变化地图**：年份滑块 + 统一色阶跨年对比的年度价格地图
- **配套设施叠加**：MRT/LRT 站点（NE14-NE17 + LRT 线路）、学校（5 所小学 + 5 所中学）
- **下方联动图表**：年度价格趋势折线图 + 年度涨跌幅热力图

### 3. 影响因素分析（03_factors.py）
分析 **7 个 Tab**，涵盖 6 个因素维度 + 策略验证：

| Tab | 分析方法 | 主要发现 |
|-----|---------|---------|
| ① 面积 | 散点图 + OLS 趋势线 + Pearson 相关 | 面积核心驱动单价，r≈0.75 |
| ② 房型 | 箱线图 + ANOVA 检验 | 房型间价差极显著（p<0.001），每级溢价 15–25% |
| ③ 楼层 | 箱线图 + 趋势分析 | 中高层溢价 5–10% |
| ④ 剩余租约 | 分箱箱线图 + 线性回归 | 租约越短折价越大 |
| ⑤ 房龄 | 散点图 + LOWESS 趋势 | 新房显著溢价，折旧非线性 |
| ⑥ 镇区对比 | Town vs Price 偏离度分析 + CAGR | 三镇区均价对比，偏离度量化地段溢价 |
| 💡 策略验证 | 4 策略 CAGR + CV + 雷达图 | 各策略适用场景与数据支撑分析 |

### 4. 价格预测（04_prediction.py）
- **特征工程**：8 个特征 —
  - 数值特征：面积、剩余租约、房龄、楼层中位、房型序数编码（2-Room=1, …, Executive=5）、MRT 距离（Haversine）、学校数量、交易年份
  - 类别特征：镇区（OneHot 编码）
  - 预测目标：**单价 price_per_sqm（新元/sqm）**
- **数据划分**：按时间切分 — 2020–2023 年训练集，2024 年及以后测试集（禁止随机划分）
- **模型选择**：Ridge Regression / Random Forest（用户可勾选）
- **超参调整**：通过 `st.slider` 交互式调节（Ridge α、RF 树数/深度）
- **分房型评估**（核心）：按 6 个类别分别计算 MAE/MAPE/R² — 小户型/中户型/大户型/老旧组屋/新近组屋/MRT 沿线
- **模型评估**：MAE、RMSE、R²、MAPE 指标卡片
- **交叉验证**：5-fold CV，报告 MAE 均值±标准差、R² 均值±标准差
- **预测 vs 实际散点图**：按房型着色，含 y=x 参考线
- **残差诊断**：残差分布直方图 + 残差 vs 预测值散点图
- **特征重要性**：RF 特征重要性（聚合）+ Ridge 标准化系数对比
- **房价预估器**：用户输入房产属性 → 多模型实时预测 → 90% 预测区间（RF 个体树分位数）→ **相似历史成交**（四级渐进式放宽匹配策略：面积±10→±30sqm，租约±5→不限，兜底同镇区+同房型）

### 5. 分析思考题（05_questions.py）
- **3 个分析 Tab**，每题含图表 + 数据分析 + st.write ≥200 字论述：
  - **Q1 — HDB 转售市场「保值性」关键因素**：面积→基本盘 / 剩余租约→时间折扣 / 镇区→地段溢价，含 CAGR 量化 + Haversine MRT 距离计算
  - **Q2 — 购房策略方向论证**：4 策略逻辑推演 + Train/Test 数据支撑 + 策略方向选择理由
  - **Q3 — 模型预测能力可靠性**：Ridge vs RF 时间切分对比 + 5-fold CV + 残差诊断 + 适用场景分析
- 仅使用 3 个东北部镇区数据（Punggol / Sengkang / Hougang）
- 技术实现：RandomForestRegressor / Ridge / StandardScaler / OneHotEncoder / ColumnTransformer

### 6. 策略验证（06_strategy.py）
- **策略设定**：4 种策略类型 —
  - 镇区偏离回归策略（买低于镇区均价 → 等待回归）
  - MRT 便利策略（买靠近 MRT → 交通溢价）
  - 长剩余租约策略（买长租约 → 减少折旧）
  - 低总价入门策略（买低总价小户型 → 降低门槛）
- **硬 2023/2024 时间切分**（防作弊）：2020–2023 策略形成期，2024+ 验证期
- **7 项验证指标**：总涨幅、年化涨幅 (CAGR)、价格波动 (CV)、年度涨幅标准差、最大回撤、成交量 CV、预算适配性 (%)
- **策略组 vs 基准组对比**：对比表格 + 季度均价走势折线图 + 关键指标差异柱状图
- **策略反思**：st.write 200+ 字（策略选择理由、优劣势、市场变化影响、改进方向）

## 四、核心分析过程

### 4.1 数据预处理技术
- 3-sigma 规则剔除价格异常值（分镇区×房型分组）
- 1%/99% 分位数截尾处理面积离群值
- 缺失值关键字段行删除
- 字符字段标准化（大小写统一、空格清理）

### 4.2 统计分析技术
- **ANOVA**：检验房型间价格差异
- **Kruskal-Wallis H 检验**：镇区间非参数检验
- **Pearson 相关系数**：面积、剩余年限与价格的线性关联
- **OLS 线性趋势**：面积/房龄对价格的影响方向和强度
- **Town vs Price 偏离度**：(镇区均价 - 三镇区基线均价) / 基线均价 × 100%
- **CAGR 年化复合增长率**：量化各镇区×房型的保值潜力

### 4.3 机器学习技术
- **Ridge Regression**：L2 正则化线性模型，处理多重共线性，可通过 `year` 特征外推价格时间趋势
- **Random Forest**：250 棵树的集成学习，捕获非线性关系和交互效应
- **5-fold 交叉验证**：评估模型泛化能力
- **特征重要性**：RF 基尼重要性 + 类别特征聚合 + Ridge 系数对比
- **预测区间**：RF 个体树预测分位数（5%/95%）
- **四级渐进式相似匹配**：面积±10→±30sqm + 租约放宽 + 兜底策略

### 4.4 购房策略量化
- **CAGR**：训练期到验证期的年化复合增长率回测
- **变异系数 CV**：价格稳定性代理指标
- **7 项验证指标**：总涨幅 / CAGR / 价格波动 CV / 年度涨幅标准差 / 最大回撤 / 成交量 CV / 预算适配性
- **策略组 vs 基准组**：硬时间切分对比验证

## 五、模型效果

预测目标：**单价 price_per_sqm（新元/sqm）**，测试集 2024 年至今。

| 模型 | MAE (新元/sqm) | MAPE | R² |
|------|:---------:|:----:|:---:|
| Ridge Regression | ~S$520/sqm | ~7.7% | ~0.48 |
| Random Forest | ~S$700/sqm | ~10.0% | ~0.19 |

- **Ridge Regression 在时间切分验证中表现优于 Random Forest**，原因是 RF 无法外推价格时间趋势（训练集 2020–2023 → 测试集 2024+），而线性模型可通过 `year` 特征线性外推
- 5-fold CV 确认 Ridge 泛化性能稳定
- 残差分布近似正态，无系统性偏差
- 特征重要性排名：**floor_area_sqm > flat_type_ordinal > remaining_lease > year**，面积仍是最强预测因子
- 分房型 MAPE 在各子类间表现稳定
- 四级渐进式相似匹配确保任何输入属性组合都能找到历史参考成交

## 八、总结与反思

### 8.1 项目总结
本项目成功构建了一个涵盖数据概览、地图可视化、因素分析、价格预测、分析思考题和策略验证六大模块的 HDB 转售价格分析与预测系统。系统基于 2020 年至今 34,914 条真实成交数据，采用统计分析和机器学习方法，提供了从宏观市场认知到微观房产定价再到购房决策支持的完整链路。

### 8.2 张涛 — 遇到的问题和个人收获

**负责模块**：数据管线、系统架构、数据概览、地图可视化、影响因素分析、分析思考题、系统部署、项目报告

**遇到的问题**：
1. **data.gov.sg API 分页限制**：CKAN API 全局查询存在约 5 万条隐形上限，导致初始只能获取 2024 年后的数据，无法满足"2020 年至今"的要求。解决方案：改为按镇区使用 `filters` 参数分别请求，每个镇区数据量在 1-2 万条之间，绕过了全局限制，成功获取完整 2020-2026 年数据。
2. **Streamlit Cloud 网络限制**：部署后发现云端无法直接访问 data.gov.sg API，导致应用报错。解决方案：将本地缓存 Parquet 文件纳入 Git 仓库，并在代码中加入 API 失败降级逻辑（优先本地缓存，API 不可用时回退）。
3. **热力图坐标精度问题**：API 数据不含精确经纬度，只有镇区名称。解决方案：手动采集各镇区 HDB 聚类中心坐标，并通过随机抖动（~300m）模拟真实空间分布。
4. **数据质量问题**：原始数据存在异常值（如极端高价/低价、面积异常）。解决方案：采用 3-sigma 分组异常值检测 + 1%/99% 分位数截尾 + 关键字段缺失行删除，确保分析数据质量。
5. **分析框架迭代**：初始采用成熟区/非成熟区二分框架，后因三镇区均为东北区域、不具代表性而重构为 Town vs Price 均价偏离度分析框架，更准确地反映镇区间的价格差异特征。

**个人收获**：
- 掌握了从公开 API 到分析系统的完整数据管线搭建流程
- 深入理解了 CKAN API 的特性与限制，学会了应对分页上限的策略
- 提升了 Streamlit 多页面应用架构设计能力
- 学会用 Plotly 和 pydeck 构建专业级交互式可视化
- 理解了统计分析（ANOVA、Pearson、Kruskal-Wallis）在真实房价分析中的应用
- 学会了根据数据特征灵活调整分析框架（Town vs Price 偏离度替代成熟/非成熟二分法）

### 8.3 申经纬 — 遇到的问题和个人收获

**负责模块**：价格预测模型、策略验证、答辩 PPT

**遇到的问题**：
1. **时间切分下的模型选择**：Random Forest 在随机 CV 中表现优于 Ridge，但在时间切分验证中表现更差——RF 无法外推价格时间趋势（树模型不会线性外推）。解决方案：采用时间切分作为主评估方法，选用 Ridge（year 特征线性外推）作为推荐模型。
2. **预测区间构建**：传统线性模型只输出点预测，无法表达不确定性。解决方案：利用 Random Forest 250 棵个体树的预测分布，取 5%/95% 分位数作为 90% 预测区间。
3. **相似历史成交匹配过严**：最初四重过滤（flat_type + town + 面积±10 + 租约±5）几乎无法匹配到结果。解决方案：设计四级渐进式放宽策略（面积±10→±30sqm，租约±5→不限），兜底模式确保总有结果返回，并标注实际使用的筛选范围。
4. **策略验证评分体系**：初始采用三维度综合评分 + 可调权重，后续按 rubric 要求改为 4 种具体策略 + 7 项独立验证指标 + 策略反思，不展示综合评分。

**个人收获**：
- 深入实践了 Ridge Regression 和 Random Forest 的模型训练、调参、评估全流程
- 理解了时间切分 vs 随机切分对模型评估的深远影响
- 学会用特征重要性分析方法理解模型的决策逻辑
- 掌握了将定量分析（CAGR、变异系数、偏离度）转化为购房决策支持的方法论
- 锻炼了用简洁语言向非技术人员传达分析结果的能力
- 学会了根据 rubric 要求灵活调整策略验证的指标体系

### 8.4 不足与改进方向
1. **地理位置数据精度**：热力分布基于镇区聚类坐标的随机抖动，而非真实经纬度；接入 OneMap API 可获精确坐标
2. **预测模型可丰富**：可增加 XGBoost、LightGBM 等模型，以及时间序列预测（Prophet/LSTM）
3. **政策冲击量化**：BSD/ABSD/HDB 资格接口已预留但未深入分析，可通过断点回归（RDD）或事件研究法量化政策影响
4. **外部因素纳入**：利率、CPI、BTO 供应量等宏观因素尚未纳入模型
5. **配套设施数据**：仅覆盖 3 镇区 MRT 距离点，不足以独立建模分析配套设施对价格的影响
