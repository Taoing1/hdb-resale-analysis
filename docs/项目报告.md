# 新加坡 HDB 组屋转售价格分析与预测系统 — 项目报告

---

## 一、项目背景

新加坡 Housing Development Board（HDB）组屋承载了超过 80% 新加坡常住人口的居住需求。组屋转售市场价格受房型、面积、剩余年限、地段、楼层等多重因素影响，对购房者而言，如何在海量交易数据中快速获取有效信息、合理评估房产价值、制定明智的购房决策，是一个具有实际意义的问题。

本项目基于 **data.gov.sg** 官方开放的 HDB 转售交易数据，选取 **Punggol、Sengkang、Hougang** 三个相邻的东北部镇区作为研究对象，构建了一个基于 Streamlit 的交互式分析与预测系统，帮助用户直观了解市场全貌、量化各因素影响、预测转售价格，并基于数据驱动的方法提出购房策略建议。

## 三、系统功能

系统共包含 **5 个功能页面**，通过 Streamlit 侧边栏导航：

### 1. 数据概览（01_overview.py）
- **多条件筛选**：镇区、年份、房型、面积、价格、剩余年限、楼层范围
- **统计看板**：交易套数、均价、中位数、总成交额、最高/最低价（6 指标卡片）
- **镇区分组统计**：各镇区套数、均价、中位数、单价、成交总额
- **趋势图表**：月度均价走势（折线图）、季度交易量（柱状图）
- **分布分析**：价格分布直方图、镇区×房型均价热力图
- **数据表格**：可配置列的交互式数据表，支持排序和 CSV 导出

### 2. 地图可视化（02_map.py）
- **镇区均价地图**：CircleMarker 展示各镇区均价，圆圈大小 = 交易量，颜色深度 = 均价
- **热力分布地图**：基于 Folium HeatMap 的房价热力分布（含坐标抖动仿真）
- **时空变化地图**：年份滑块 + 同比涨跌幅标注的年度价格地图
- **叠加图层接口**（预留）：MRT 站点（LTA DataMall）、学校（data.gov.sg School API）、商场（OSM Overpass/OneMap）

### 3. 影响因素分析（03_factors.py）
分析 **4 个核心影响因素**：

| 因素 | 分析方法 | 主要发现 |
|------|---------|---------|
| 房型 | 箱线图 + ANOVA 检验 | 房型间价差极显著（p<0.001），每级溢价 15–25% |
| 面积 | 散点图 + OLS 趋势线 + Pearson 相关 | r≈0.75，最强单一预测因子 |
| 剩余年限 | 分箱箱线图 + 线性回归 | 每 10 年剩余年限约影响 S$30,000–60,000 |
| 镇区 | 小提琴图 + Kruskal-Wallis 检验 | 镇区间溢价约 ±5–15% |

- **高级分析**（可选）：交互效应热力图、偏效应 LOWESS 视图
- **综合结论表**：四因素联合 R² 解释力

### 4. 价格预测（04_prediction.py）
- **2 个预测模型**：
  - Ridge Regression（岭回归）— 线性基线
  - Random Forest Regressor（随机森林，250 棵树，max_depth=18）
- **预处理**：StandardScaler（数值特征）+ OneHotEncoder（类别特征）
- **模型评估**：MAE、RMSE、R² 指标卡片 + 柱状图对比
- **交叉验证**：5-fold CV，报告 MAE 均值±标准差、R² 均值±标准差
- **可视诊断**：预测值 vs 实际值散点图、残差分布直方图、残差 vs 预测值散点图
- **特征重要性**：RF 特征重要性（聚合）+ Ridge 标准化系数
- **交互式预测**：用户输入房产属性 → 双模型实时预测 → 90% 预测区间（RF 个体树分位数）→ 相似历史成交参考

### 5. 购房策略（05_strategy.py）
- **约束输入**：预算范围、目标房型、面积区间、三维度权重
- **训练/测试集时间切分**（75% 分位数年份）
- **三维度量化评分**：
  - **涨幅**：训练期均价 → 测试期均价 CAGR 回测
  - **稳定性**：变异系数 CV（越小越稳定），小样本惩罚
  - **流动性**：成交量归一化
- **综合排名**：加权综合评分 + Top 12 柱状图 + 雷达图（Top 6）
- **策略推荐**：首选推荐（含当前市场匹配房源统计）+ 备选方案
- **购房能力测算**：首付、贷款额、月供、CPF 覆盖后现金支出
- **政策影响分析**（接口预留）：BSD/ABSD/LTV/MSR/MOP 等政策因素

## 四、核心分析过程

### 4.1 数据预处理技术
- 3-sigma 规则剔除价格异常值（分镇区×房型分组）
- 1%/99% 分位数截尾处理面积离群值
- 缺失值关键字段行删除
- 字符字段标准化（大小写统一、空格清理）

### 4.2 统计分析技术
- **ANOVA**：检验房型间价格差异
- **Kruskal-Wallis H 检验**：镇区间非参数检验
- **Pearson 相关系数**：面积、剩余年限与价格的线性关联
- **OLS 线性趋势**：面积/房龄对价格的影响方向和强度

### 4.3 机器学习技术
- **Ridge Regression**：L2 正则化线性模型，处理多重共线性
- **Random Forest**：250 棵树的集成学习，捕获非线性关系和交互效应
- **5-fold 交叉验证**：评估模型泛化能力
- **特征重要性**：RF 基尼重要性 + 类别特征聚合 + Ridge 系数对比
- **预测区间**：RF 个体树预测分位数（5%/95%）

### 4.4 购房策略量化
- **CAGR**：训练期到测试期的年化复合增长率回测
- **变异系数**：价格稳定性代理指标
- **多维加权评分**：可调权重的综合排名

## 五、模型效果

| 模型 | MAE（SGD） | RMSE（SGD） | R² |
|------|:---------:|:----------:|:---:|
| Ridge Regression | ~S$45,000 | ~S$62,000 | ~0.85 |
| Random Forest | ~S$28,000 | ~S$40,000 | ~0.94 |

- Random Forest 在各项指标上均优于 Ridge Regression
- 5-fold CV 确认 RF 的泛化性能稳定（R² CV ≈ 0.93）
- 残差分布近似正态，无系统性偏差
- 特征重要性排名：**floor_area_sqm > remaining_lease > town > flat_type**，面积是最强预测因子

## 八、总结与反思

### 8.1 项目总结
本项目成功构建了一个涵盖数据概览、地图可视化、因素分析、价格预测和购房策略五大模块的 HDB 转售价格分析与预测系统。系统基于 2020 年至今 34,914 条真实成交数据，采用统计分析和机器学习方法，提供了从宏观市场认知到微观房产定价再到购房决策支持的完整链路。

### 8.2 张涛 — 遇到的问题和个人收获

**负责模块**：数据管线、系统架构、数据概览、地图可视化、影响因素分析、系统部署

**遇到的问题**：
1. **data.gov.sg API 分页限制**：CKAN API 全局查询存在约 5 万条隐形上限，导致初始只能获取 2024 年后的数据，无法满足"2020 年至今"的要求。解决方案：改为按镇区使用 `filters` 参数分别请求，每个镇区数据量在 1-2 万条之间，绕过了全局限制，成功获取完整 2020-2026 年数据。
2. **Streamlit Cloud 网络限制**：部署后发现云端无法直接访问 data.gov.sg API，导致应用报错。解决方案：将本地缓存 Parquet 文件纳入 Git 仓库，并在代码中加入 API 失败降级逻辑（优先本地缓存，API 不可用时回退）。
3. **热力图坐标精度问题**：API 数据不含精确经纬度，只有镇区名称。解决方案：手动采集各镇区 HDB 聚类中心坐标，并通过随机抖动（~300m）模拟真实空间分布。
4. **数据质量问题**：原始数据存在异常值（如极端高价/低价、面积异常）。解决方案：采用 3-sigma 分组异常值检测 + 1%/99% 分位数截尾 + 关键字段缺失行删除，确保分析数据质量。

**个人收获**：
- 掌握了从公开 API 到分析系统的完整数据管线搭建流程
- 深入理解了 CKAN API 的特性与限制，学会了应对分页上限的策略
- 提升了 Streamlit 多页面应用架构设计能力
- 学会用 Plotly 和 Folium 构建专业级交互式可视化
- 理解了统计分析（ANOVA、Pearson、Kruskal-Wallis）在真实房价分析中的应用

### 8.3 申经纬 — 遇到的问题和个人收获

**负责模块**：价格预测模型、购房策略分析、答辩 PPT

**遇到的问题**：
1. **模型特征共线性**：面积和房型高度相关，同时在模型中可能导致系数不稳定。解决方案：Ridge Regression 的 L2 正则化有效缓解了多重共线性问题，同时在 RF 特征重要性中做了类别特征聚合分析。
2. **预测区间构建**：传统线性模型只输出点预测，无法表达不确定性。解决方案：利用 Random Forest 250 棵个体树的预测分布，取 5%/95% 分位数作为 90% 预测区间。
3. **购房策略回测设计**：如何用历史数据验证策略有效性。解决方案：采用 Train/Test 时间切分（前 75% 年份 → 后 25% 年份），在训练期计算三维度评分，测试期验证实际表现（CAGR）。
4. **三维度权重设定**：涨幅、稳定性、流动性三者的相对重要性因购房目的而异。解决方案：设计可交互调节的权重滑块（实时归一化），并提供分场景的策略建议（自住/投资/短期/长期）。

**个人收获**：
- 深入实践了 Random Forest 和 Ridge Regression 的模型训练、调参、评估全流程
- 理解了交叉验证和残差诊断在模型验证中的重要性
- 学会用特征重要性分析方法理解模型的决策逻辑
- 掌握了将定量分析（CAGR、变异系数）转化为购房决策支持的方法论
- 锻炼了用简洁语言向非技术人员传达分析结果的能力

### 8.4 不足与改进方向
1. **地理位置数据精度**：热力分布基于镇区聚类坐标的随机抖动，而非真实经纬度；接入 OneMap API 可获精确坐标
2. **叠加图层未接入**：MRT/学校/商场图层接口已预留但需 API Key 方可实际调用
3. **预测模型可丰富**：可增加 XGBoost、Gradient Boosting 等模型，以及时间序列预测（Prophet/LSTM）
4. **政策冲击量化**：当前仅为接口预留，可通过断点回归（RDD）或事件研究法量化政策影响
5. **外部因素纳入**：利率、CPI、BTO 供应量等宏观因素尚未纳入模型

### 8.5 免责声明
本系统房价预测结果仅用于课程学习目的，**不构成任何购房投资建议**。实际购房请咨询持牌专业人士。
