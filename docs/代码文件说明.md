# HDB 转售价格分析系统 — 代码文件说明

## 项目概述

基于 Streamlit 的新加坡 HDB 组屋转售价格分析与预测系统，覆盖 Punggol、Sengkang、Hougang 三个镇区。共 6 个功能页面，采用 pydeck 地图可视化、Plotly 交互式图表、Scikit-learn 机器学习模型。


## 文件功能详解

### app.py — 主入口
- 设置 Streamlit 页面配置（标题、图标、宽屏布局）
- 通过 importlib 动态加载 6 个功能页面模块
- 侧边栏导航（radio 选择器）
- 调用 `load_data()` 加载数据 → 传递给当前页面的 `run(df)` 函数

### data/data_loader.py — 数据管线
- `fetch_hdb_data(force_refresh)`: 从 data.gov.sg API 分页获取数据，本地 Parquet 缓存，API 失败时降级使用缓存
- `clean_hdb_data(df)`: 12 步数据清洗流程（镇区筛选、日期解析、数值转换、剩余年限计算、房型标准化、楼层解析、单价计算、异常值剔除等）
- `load_data(force_refresh)`: 完整管线 fetch → clean

### data/hdb_cache.parquet — 数据缓存
- 内置 HDB 转售数据缓存，清洗后目标镇区 34,914 条
- 确保 Streamlit Cloud 部署时无需网络也可运行

### pages/01_overview.py — 数据概览
- `_build_sidebar_filters()`: 9 项筛选控件（全选/清除快捷按钮）
- `_apply_filters()`: 多条件组合筛选
- `_render_stats_dashboard()`: 8 指标卡片 + 镇区分组统计表
- `_render_trend_chart()`: 月度均价走势（Plotly 折线图）
- `_render_volume_chart()`: 季度交易量（Plotly 柱状图）
- `_render_price_distribution()`: 价格分布直方图
- `_render_town_type_matrix()`: 镇区×房型 均价热力图
- `_render_data_table()`: 可配置列的交互式数据表 + CSV 导出

### pages/02_map.py — 地图可视化（pydeck）
- `_build_town_avg_map()`: 镇区均价 ScatterplotLayer 地图（圆圈大小=交易量，颜色=均价）
- `_build_heatmap()`: 房价热力图 HeatmapLayer（含坐标抖动仿真 + 可调半径/强度）
- `_build_temporal_map()`: 年份滑块地图（统一色阶跨年对比）
- `_add_overlay_layers()`: MRT 站点 + 学校 ScatterplotLayer 叠加（侧边栏多选控制）
- `_render_temporal_chart()`: 年度价格趋势线图 + 涨跌幅热力图
- `_render_town_stats_table()`: 镇区分组统计表
- 内置 MRT_STATIONS（9 个站点：NE13-NE17 + LRT 线路）和 SCHOOLS（5 小学 + 5 中学）坐标数据

### pages/03_factors.py — 影响因素分析
- **7 个分析 Tab**：① 面积 / ② 房型 / ③ 楼层 / ④ 剩余租约 / ⑤ 房龄 / ⑥ 镇区对比 / 💡 策略验证
- `_analyze_area()`: 单价 vs 面积散点图 + OLS 趋势线 + Pearson 相关系数
- `_analyze_flat_type()`: 总价箱线图 + ANOVA 检验 + 逐级溢价分析
- `_analyze_storey()`: 楼层箱线图 + 趋势分析
- `_analyze_lease()`: 分箱箱线图 + 散点图 + 折旧斜率
- `_analyze_age()`: 房龄 vs 价格 LOWESS 趋势
- `_analyze_town_comparison()`: Town vs Price 偏离度分析 + CAGR（替代旧版成熟/非成熟对比框架）
- `_analyze_strategy()`: 4 策略回测 + 季度趋势 + 雷达图 + 策略说明
- 仅使用三镇区数据（TARGET_TOWNS = ["PUNGGOL", "SENGKANG", "HOUGANG"]）

### pages/04_prediction.py — 价格预测
- **特征工程（8 特征）**：房龄、房型序数编码（2-Room=1 → Executive=5）、MRT 距离（Haversine 公式）、学校数量、剩余租约、面积、楼层中位、年份 + town（OneHot 编码）
- 预测目标：**单价 price_per_sqm（新元/sqm）**
- **硬时间切分**：2020–2023 训练集 / 2024+ 测试集（禁止随机划分）
- 模型选择：Ridge / RandomForest（侧边栏勾选）
- **超参调整**：st.slider 交互调节（alpha、n_estimators、max_depth）
- **分房型评估**（核心）：6 类别分别计算 MAE/MAPE/R²
  - 小户型（2-Room, 3-Room）
  - 中户型（4-Room）
  - 大户型（5-Room, Executive）
  - 老旧组屋（房龄 ≥ 35 年）
  - 新近组屋（房龄 < 15 年）
  - MRT 沿线（MRT 距离 < 500m）
- 模型评估：MAE / RMSE / R² / MAPE 指标卡片
- 交叉验证：5-fold CV，报告 MAE 均值±标准差、R² 均值±标准差
- 预测 vs 实际散点图（按房型着色）+ y=x 参考线
- 残差诊断：残差分布直方图 + 残差 vs 预测值散点图
- 特征重要性：RF 基尼重要性（聚合）+ Ridge 标准化系数对比
- 交互式预测表单：用户输入 → 多模型预测 → 90% 预测区间（RF 个体树分位数）
- **四级渐进式相似成交匹配**：
  - `_find_similar()` 函数：面积±10→±15→±20→±30sqm + 租约放宽 + 兜底同镇区+同房型
  - 返回使用何种筛选范围和 Top 10 最近似交易

### pages/05_questions.py — 分析思考题
- **3 个 Tab**，每题含图表 + 数据分析 + st.write ≥200 字论述
- **Q1 — 保值性分析**：
  - 面积—价格基本盘 / 剩余租约—时间折扣 / 镇区—地段溢价三层框架
  - CAGR 年化复合增长率计算（分面积/房龄/镇区维度）
  - Haversine 公式计算 MRT 距离
- **Q2 — 策略方向论证**：
  - 4 策略逻辑推演：镇区偏离回归 / MRT 便利 / 长剩余租约 / 低总价入门
  - Train/Test 数据支撑分析
  - 策略选择理由论述
- **Q3 — 模型可靠性分析**：
  - Ridge vs RF 时间切分对比
  - 5-fold CV 验证
  - 残差诊断 + 适用场景 + 模型局限性
- 仅使用 3 镇区数据，不引入外部数据源
- 技术栈：RandomForestRegressor / Ridge / StandardScaler / OneHotEncoder / ColumnTransformer

### pages/06_strategy.py — 策略验证
- **4 种策略类型**：
  - 镇区偏离回归策略：买低于镇区均价 → 等待价格回归
  - MRT 便利策略：买靠近 MRT → 享受交通溢价
  - 长剩余租约策略：买租约最长 → 减少折旧损失
  - 低总价入门策略：买低总价小户型 → 降低入场门槛
- **硬 2023/2024 时间切分**（防作弊）：2020–2023 策略形成期，2024+ 验证期
- 策略设定：策略类型选择 + 预算上限 + 户型范围 + 候选镇区
- 选择规则随策略类型而变：价格偏离度最小/MRT 距离最近/剩余租约最长/总价最低
- **7 项验证指标**：总涨幅 / CAGR / 价格波动 CV / 年度涨幅标准差 / 最大回撤 / 成交量 CV / 预算适配性(%)
- 策略组 vs 基准组：对比表格 + 季度均价走势折线图 + 关键指标差异柱状图
- **策略反思**：st.write 200+ 字（策略选择理由 / 优劣势 / 市场变化影响 / 改进方向）

### utils/helpers.py — 工具函数
- `TOWN_COLORS`: 三镇区配色映射（Plotly 图表用）
- `TOWN_COORDS`: 三镇区经纬度坐标（地图用）
- `fmt_price()`: 新元金额格式化（K/M 单位）
- `fmt_pct()`: 百分比格式化
- `get_model_metrics()`: 计算 MAE / RMSE / R²

## 架构说明

### 页面导航机制
`app.py` 使用 `importlib.import_module()` 动态加载 pages/ 目录下的页面模块，每个页面暴露一个 `run(df)` 函数。侧边栏 radio 选择器切换页面时，仅调用对应模块的 `run()` 函数。这种设计：
- 避免一次导入所有页面（减少启动时间）
- 页面间完全解耦（各自独立维护）
- 通过 `df` 参数传递共享数据（数据只加载一次）

### 数据流
```
data.gov.sg API
      ↓
data_loader.fetch_hdb_data()  ←→  hdb_cache.parquet
      ↓
data_loader.clean_hdb_data()
      ↓
load_data() @st.cache_data
      ↓
app.py → page.run(df)
```

### 防止数据泄露（策略验证）
策略验证页面采用硬时间切分：
- `train_df = df[df["year"] <= 2023]` — 策略形成期
- `test_df = df[df["year"] >= 2024]` — 策略验证期
- 策略定义仅使用训练期数据，验证仅使用测试期数据

### Town vs Price 偏离度框架
替代传统的成熟/非成熟二分法，采用：
- `偏离度 = (镇区均价 - 三镇区基线均价) / 基线均价 × 100%`
- 三镇区基线均价 = Punggol / Sengkang / Hougang 的等权均值
- CAGR（年化复合增长率）作为保值潜力代理指标
