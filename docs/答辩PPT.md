## 第 2 页：项目背景

**为什么做这个项目？**
- 新加坡超 80% 人口居住在 HDB 组屋，转售市场活跃
- 房价受多重因素影响：房型、面积、屋龄、地段、楼层…
- 购房者面对海量数据，缺乏直观分析工具

**我们要解决什么问题？**
- 如何快速了解某个区域的房价全貌？
- 哪些因素对房价影响最大？
- 能否用模型预测一套房的价格？
- 预算有限的情况下，买哪里、买什么房型最划算？

**研究对象**
- 3 个相邻镇区：Punggol（榜鹅）、Sengkang（盛港）、Hougang（后港）
- 数据来源：data.gov.sg 官方 HDB 转售交易
- 时间范围：2020 年至今
- 记录数：**约 33,000+ 条**（清洗后）

## 第 6 页：影响因素分析 — 4 个核心因素

| 因素 | 方法 | 发现 |
|------|------|------|
| ① 房型 | 箱线图 + ANOVA | p<0.001，每级溢价 15-25% |
| ② 面积 | 散点图 + Pearson r | r≈0.75，最强单一因子 |
| ③ 剩余年限 | 分箱箱线图 + 线性回归 | 每 10 年差 S$30K-60K |
| ④ 镇区 | 小提琴图 + Kruskal-Wallis | 镇区溢价 ±5-15% |

**高级分析**
- 交互效应：镇区 × 房型密度热力图
- 偏效应：面积 / 房龄 / 楼层 LOWESS 趋势

**四因素联合 R² ≈ 0.80**

## 第 7 页：价格预测 — 双模型对比

| | Ridge Regression | Random Forest |
|------|:---:|:---:|
| 类型 | 线性（L2 正则） | 集成树（250 棵树） |
| MAE | ~S$45K | **~S$28K** |
| RMSE | ~S$62K | **~S$40K** |
| R² | ~0.85 | **~0.94** |
| CV R²（5-fold） | ~0.84 | **~0.93** |

**Random Forest 表现更优**：能捕获非线性关系和交互效应

**诊断**
- 预测值 vs 实际值散点图（误差率着色）
- 残差分布近似正态、无系统性偏差

## 第 8 页：特征重要性

**Random Forest 特征重要性排名：**

| 排名 | 特征 | 重要性 |
|:---:|------|:---:|
| 1 | floor_area_sqm（面积） | ★★★★★ |
| 2 | remaining_lease（剩余年限） | ★★★★ |
| 3 | year（交易年份） | ★★★ |
| 4 | storey_mid（楼层） | ★★★ |
| 5 | town（镇区） | ★★ |
| 6 | flat_type（房型） | ★ |

**结论**：面积是最强预测因子，房型/镇区被面积部分替代（面积和房型高度相关）

**Ridge 系数（标准化后）**：与 RF 重要性排名高度一致

## 第 10 页：购房策略 — 三维度量化

**约束条件**：预算、房型、面积

**三维度评分体系**：

| 维度 | 指标 | 计算方式 |
|------|------|---------|
| 涨幅 | 历史增值潜力 | Train/Test 时间切分 CAGR 回测 |
| 稳定性 | 价格波动风险 | 变异系数 CV（越小越稳） |
| 流动性 | 交易活跃度 | 分镇区×房型成交量归一化 |

**可调权重 + 综合排名**

---

## 第 11 页：策略回测与推荐

**Train/Test 时间切分**：前 75% 年份训练 → 后 25% 年份验证

**推荐输出**：
- 首选推荐（镇区 + 房型 + 三维度评分 + 当前匹配房源统计）
- 备选 Top 3
- 雷达图（Top 6 多维度可视化）

**购房能力测算**
- 首付 / 贷款额 / 月供 / CPF 覆盖

**策略建议**
- 自住：优先稳定性
- 投资：优先涨幅
- 短期：加权流动性
- 长期：加权涨幅 + 面积

## 第 12 页：技术亮点

1. **完整数据管线**：API → 缓存 → 自动清洗 → 3-sigma/分位数异常值处理
2. **双模型对比**：线性（可解释）+ 集成树（高精度），互补分析
3. **预测区间估计**：RF 个体树分位数 → 90% 置信区间
4. **三维度策略量化**：涨幅（CAGR）+ 稳定性（CV）+ 流动性（成交量）
5. **可扩展架构**：MRT/学校/商场/PBSD/ABSD 等接口均已预留
6. **一键部署**：内置缓存数据，Streamlit Community Cloud 免费部署

## 第 13 页：项目挑战与解决

| 挑战 | 解决方案 |
|------|---------|
| API 数据量大（23 万条） | 本地 Parquet 缓存 + 分页获取仅目标数据 |
| 数据质量问题（缺失/异常） | 关键字段行删除 + 3-sigma 分组异常值剔除 + 分位数截尾 |
| 热力图无真实坐标 | 镇区聚类中心坐标 + 随机抖动（~300m 范围） |
| 模型过拟合 | 5-fold CV 验证 + RF 限制 max_depth/min_samples_leaf |
| Streamlit Cloud 无法访问 API | 内置缓存 Parquet 文件 + API 失败降级策略 |
| 多页面代码组织 | Streamlit pages/ 目录 + importlib 动态导入 |
